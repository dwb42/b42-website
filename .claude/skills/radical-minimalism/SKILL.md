# radical-minimalism

## Purpose
Prevent overengineering and enforce extreme restraint. The site is text-only. Absence is the design.

## When to use
- Before adding any new element, section, component, or line of copy.
- When reviewing a diff for scope creep.
- When the instinct is "this page needs something more" — use this skill to resist that instinct.
- When deciding between two options, to pick the smaller one.

## Principles
- Do not add anything not explicitly requested.
- If something feels missing, leave it missing.
- Remove instead of add.
- Silence is a feature.
- Every element must justify its existence against deletion.
- Restraint compounds; each omission strengthens the next.

## Instructions
- Never introduce UI patterns (nav bars, headers, footers, menus, modals, tabs).
- Never add buttons, links, or navigation unless the user asked for them by name.
- Never add sections, dividers, or scaffolding "for structure."
- Never add helper text, captions, labels, or instructions.
- Never explain more than necessary in copy or in code comments.
- When in doubt, ship less. When still in doubt, ship nothing.
- If the user asks for X, deliver exactly X — not X plus obvious companions.

## Constraints
- No CTAs.
- No helper text, tooltips, or hints.
- No placeholders or lorem ipsum.
- No marketing language, taglines, or value props unless provided verbatim.
- No logos, badges, or social proof.
- No footers, copyright lines, or metadata blocks unless requested.
- No analytics, cookie banners, or consent UI unless requested.
